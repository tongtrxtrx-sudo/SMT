# SMT Guard 搜索、比较与布局验收记录

## 目标

- 扫码作业的产品配置支持输入产品编码或版本片段查询，并要求从候选项选择有效配置。
- 生产运行的“查询运行”按钮紧邻运行号、操作员和状态筛选，不再被空白区域推到最右侧。
- BOM 版本比较的第二个候选框只显示与第一个版本相同产品的其他版本。
- 全部业务表格支持拖动列宽和列顺序；BOM、产品配置、生产运行、设备站位和审计日志的
  左右板块支持拖动比例；所有布局在重启后恢复。
- BOM 和产品配置的筛选卡片保持 760–980 像素并左对齐，不再横跨整个工作区。
- 左侧“作业 / 配置 / 系统”使用独立灰底和左侧标识，功能入口缩进显示。
- 全部业务表格中，被当前列宽截断的字段在点击后显示完整内容。

## 持久化位置

布局写入应用数据目录的 `ui_layout.json`。文件只保存 Qt 表头和分栏状态，不保存或复制
BOM、扫码记录等业务内容。表格初次打开仍按全屏规则分配宽度；只有用户主动拉宽列后，
横向滚动条才按需出现。

## 自动化覆盖

- 扫码配置框为可编辑下拉框，补全规则为不区分大小写的包含匹配；非完整候选不能开始运行。
- 多产品 BOM 同时存在时，第二比较框不出现其他产品；只有单个版本的产品禁用比较按钮。
- 生产运行查询按钮文案和位置明确，回车查询仍保留。
- 表头移动、列宽调整和分栏移动写入 `ui_layout.json`，新建控件模拟重启后恢复相同状态。
- 公共表格控制器检测文字是否被截断，仅对超长字段的单元格点击显示完整内容。
- BOM 和产品配置筛选卡片最大宽度，以及既有导航组合均有场景测试保护。
