# SMT Guard 全屏界面优化验收记录

## 1. 结论

本轮已按现场全屏显示目标优化全部 8 个页面，同时保留 1180 x 760 中等窗口兼容性。根据现场截图反馈继续完成第四轮统一美化：取消扫码页左右栏，主任务、扫码输入、单行进度与全宽扫码记录按现场阅读顺序纵向排列；上次确认的操作员在下一次启动时自动恢复；全部管理页统一为浅色画布、白色卡片、标题蓝色强调线、状态反馈条和 36 像素表格行高。

## 2. 页面调整

- 扫码作业：取消会压缩记录字段的左右分栏。全屏下主任务卡不高于 400 像素，扫码输入高度 56 像素、字号 18 像素，进度缩为不高于 72 像素的单行横条，扫码记录默认展开并使用整页宽度与剩余高度。中等窗口主任务进一步缩短，并默认折叠记录。
- 操作员：确认后写入应用数据目录的 `last_operator.txt`；重新启动时自动恢复并保持顶部收缩状态，点击“切换”才重新展开输入框。
- 生产运行：保留现有全屏详情效果，并在页面说明中明确其用途是供班组长查看每次扫码作业的进度、NG、站位记录，以及恢复、中断和导出。
- 记录查询：运行号输入不再无边界拉伸，结果主表的重要字段均分可用宽度，结果/重复保持紧凑。
- 设备与站位：左右面板按 1:1 使用全屏宽度，设备、站位名称等主要列扩展，状态和引用标识保持短列。
- 设备与站位批量创建：原生数字微型箭头替换为与输入框同高的显式“－ / ＋”按钮，点击区域为 38 x 38 像素，支持单击和长按连续调整；避免全屏现场使用时箭头过小、图标空白或误以为按钮无效。
- 站位编码排序：列表与筛选结果改为自然数字排序，`F-1、F-2、F-10` 不再按文本顺序显示为 `F-1、F-10、F-2`；混合前缀编码（如 `R-2-A、R-10-A`）同样按数字大小排列。
- 导入配置：三步向导在全屏居中，最大宽度 1280 像素，避免文件输入区域横跨整个屏幕。
- BOM 管理：版本列表与详情按 4:6 分配，详情区不再限制为 580 像素；物料字段均衡利用横向空间。取消容易误解且不影响扫码的手动启用/停用，改为“当前版本/历史版本”；完成三步导入配置后关联 BOM 自动成为当前版本，生产可用性只由产品配置控制。
- 产品配置：配置列表与站位物料关系按 4:6 分配，版本输入限制合理宽度，右侧编辑区获得主要空间。列表直接显示产品名称、规格和可读 BOM 版本；已启用/历史版本显示不可编辑原因及“复制为草稿并编辑”入口，草稿版本开放新增行、删除行和保存修改。
- 审计日志：日志列表与变更详情按 6:4 分配，详情区不再限制为 400 像素，长实体键及变更内容可以直接阅读。
- 全局：操作员输入和常用筛选输入设置可读最大宽度；表格不再把全部剩余宽度交给最后一列；应用启动即最大化。
- 全局美化：输入框和按钮统一圆角与聚焦边框，表格使用浅色表头和交替行，内容卡片统一边框与留白；成功、提示、错误改为整条反馈区域，避免只靠文字颜色表达状态。
- 导航收敛：隐藏与“生产运行”重复的“记录查询”入口，生产运行统一承载进度、站位、NG/重复扫码记录和 CSV 导出；高级记录页仍在内部保留，确保兼容既有查询流程。
