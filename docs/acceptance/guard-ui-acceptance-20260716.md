# SMT Guard 界面优化实际 EXE 验收记录

## 1. 结论

本轮按已确认的现场扫码界面草图完成核心作业流程，并继续完成日期快捷筛选、可行动空状态、管理表格紧凑化和全界面信息层级统一。在 Windows 实际 EXE 中已走通配置导入、扫码 NG、生产运行详情、CSV 导出、空状态直达导入、日期快捷筛选，并逐页复验全部导航入口。最终复验未发现阻断问题，可以作为本地现场试用候选包继续验证。

本轮有意不扩展分页、模型视图等第三优先级工作，重点减少现场步骤、突出当前任务并隐藏低频信息。

## 2. 实现范围

- 扫码页：大号当前步骤提示；设备蓝、站位橙、物料绿；OK/NG 大面积反馈；扫码枪就绪/失焦状态；Enter 自动提交；手动提交降级；最近历史默认折叠；物料信息按步骤显示。
- 操作员：确认后收缩为当前操作员和“切换”，避免误改并释放页面高度。
- 导航：使用左侧“作业 / 配置 / 系统”分组导航，扫码保持默认主入口。
- 动作颜色：主操作蓝、启用/成功绿、停用/中断红、普通辅助操作白底描边、不可用灰。
- 导入配置：改为“导入 BOM -> 导入站位表 -> 校验并启用”三步引导，并显示产品、版本、物料数、站位数和校验结果。
- 生产运行详情：直接查看站位进度和扫码记录，显示 NG/重复统计并导出所选运行 CSV。
- 时间查询：生产运行和审计日志使用日期时间选择控件，默认近 7 天，并提供“今天 / 近 7 天 / 近 30 天”快捷查询，不再显示 ISO 技术术语。
- 可行动空状态：扫码、BOM 和产品配置无数据时直接提供导入入口；扫码无可用配置时禁用不可执行的开始控件。
- 紧凑表格：BOM 版本列表只保留 5 个高频字段，来源、SHA-256 和导入人移入右侧详情；生产运行列表只保留 6 个高频字段，结束时间和中断原因移入详情；BOM、物料、运行和默认站位进度表在验收窗口中不再出现横向滚动条。
- 刷新反馈：生产运行和审计查询结果显示“更新于 HH:mm”，让用户确认当前数据是否已刷新。
- 全界面统一：记录查询、设备与站位、BOM、产品配置、生产运行、导入配置和审计日志统一使用“页标题与说明 + 筛选/列表卡 + 详情/动作卡”的层级；低频字段进入详情区，空表改为可理解的空状态。
- 审计详情：实体与动作显示中文业务名称，原始代码保留在悬停提示中；变更前后 JSON 进入右侧详情，主表不再横向滚动。
- 保留并纳入验证：本分支原有的时间显示、中文语音、扫码引导、状态两态化和按钮互斥修改。

## 5. 实际 EXE 验收

实际窗口尺寸为 1183 x 791，语音在验收时通过环境变量关闭，避免影响自动输入。验收过程使用真实窗口点击、键盘输入和 Enter 提交，不通过测试替身绕过 UI。

### 5.1 全流程

1. 确认 `OP-ACCEPT` 后，顶部编辑区收缩为当前操作员和“切换”。
2. 在三步导入页导入 `bom-v1.xlsx` 和 `station-v1.xlsx`，界面显示产品 `90010001`、配置 `UI-V1`、6 个物料和 5 个站位，最终校验并启用成功。
3. 扫码页以 Enter 依次提交 `MNT-A`、`F-01`、`WRONG-MATERIAL`；出现大面积红色“对料错误”，同时突出要求物料 `01005R10K` 和扫码物料。
4. 最近扫码记录初始折叠，展开后显示 1 条 NG 记录。
5. 生产运行详情直接切换到扫码记录，显示“1 条 / NG 1 / 重复 0”，无需复制运行编号。
6. 从详情页导出 CSV；文件为 258 字节，中文表头及 NG 记录与 SQLite 一致。

### 5.2 最终焦点修复复验

首轮实际 EXE 验收发现：NG 后输入框已获得焦点，但状态牌短暂保留黄色“未激活”。修复后重新构建 `r2` 包，并以 `OP-R2` 再次通过 Enter 提交 `MNT-A`、`F-01`、`R2-WRONG`。

最终窗口同时显示：

- 绿色“扫码枪已就绪”；
- 大面积红色“对料错误”；
- 当前物料步骤；
- 要求物料 `01005R10K` 与扫码物料 `R2-WRONG`。

关闭窗口后，隔离数据库 `PRAGMA integrity_check` 为 `ok`；最新运行因关闭窗口按设计记录为 `INTERRUPTED`，最新尝试为 `NG / R2-WRONG`。

### 5.3 日期筛选与空状态复验

第二阶段重新构建 `r3` 包，并使用全新的隔离数据目录启动真实 EXE：

1. 扫码页未检测到可用配置时，“开始新运行”和配置下拉框不可执行，中央显示“没有可用配置，请前往导入配置”，同时提供同名蓝色入口。
2. 单击该入口后直接选中左侧“导入配置”，落在“① 导入 BOM”步骤，无需用户理解页面结构。
3. 生产运行页显示可编辑的开始/结束日期时间控件和“今天 / 近 7 天 / 近 30 天”；默认范围为 `2026-07-10 00:00` 至 `2026-07-16 23:59`。
4. 单击“今天”后，开始时间立即变为 `2026-07-16 00:00`，查询同步刷新为 0 条。
5. 审计日志页使用相同控件和快捷范围，首次进入自动查询；界面不再出现“ISO”。
6. 在 1183 x 791 实际窗口中，日期行、快捷按钮、表格和底部操作均无重叠，关键控件未截断。

关闭窗口后进程正常退出，隔离数据库 `PRAGMA integrity_check` 为 `ok`。

### 5.4 紧凑表格与更新时间复验

第三阶段使用带既有 BOM、站位、运行和 NG 扫码记录的隔离数据库构建并启动真实 EXE。首个候选包在全局样式下仍显示数像素宽的横向滚动条，随后对高频表格显式收敛列宽并关闭横向滚动，再构建最终 `r6` 包复验：

1. BOM 版本列表显示“产品 / 版本 / 状态 / 物料数 / 导入时间”5 列；来源文件、SHA-256、导入时间和操作员完整显示在右侧详情。
2. BOM 版本列表和右侧 6 行物料表均无横向滚动条，页面底部启用、停用和比较操作无重叠。
3. 生产运行列表显示“运行号 / 产品与版本 / 操作员 / 状态 / 进度 / 开始时间”6 列；右侧直接显示开始时间、结束/中断时间和中断原因。
4. 默认“站位进度”子表收敛为 5 列并消除横向滚动条；较长物料编码按单元格宽度省略，不影响行选择和完整数据来源。
5. 生产运行底部显示“找到 2 个运行 · 更新于 10:21”，审计日志使用相同更新时间反馈。

最终窗口尺寸为 1183 x 791。窗口关闭后进程正常退出，隔离数据库 `PRAGMA integrity_check` 为 `ok`；最终 EXE SHA-256 与本报告记录一致。
