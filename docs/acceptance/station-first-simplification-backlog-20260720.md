## 背景

目前要求所有设备之间的站位编码全局唯一。因此，扫码流程可以只根据站位编码识别所属设备，
同时在内部继续保留 `device_code`，用于保证产品配置完整性、生产历史追溯、审计和导出。

面向操作员的流程只应保留扫码防错真正需要的概念。已经确认 BOM 管理不参与实际的站位与物料
比对，因此对当前操作没有必要。较长的技术运行编号有利于系统内部追溯，但不适合在车间日常沟通
中突出显示。

## 已确认的目标流程

日常操作应简化为：

`维护设备和站位 -> 导入产品配置 -> 开始扫码 -> 查看作业记录`

- 产品配置由“产品编码”和“配置版本”共同确定。
- 产品配置导入表只包含“站位编码”和“物料编码”。
- 系统根据全局唯一的站位编码自动识别所属设备。
- BOM 导入、BOM 生命周期管理和 BOM 物料成员校验不属于操作员流程。
- 操作员看到短“作业号”；原有完整运行编号继续在内部保留，用于追溯。

## 建议实施范围

### 优先级 1：从操作流程中移除 BOM

- 从主导航中移除“BOM 管理”。
- 将原来的“导入 BOM -> 导入站位表 -> 校验并启用”向导改为一个“导入产品配置”流程。
- 在导入页面直接填写“产品编码”和“配置版本”，不再从 BOM 文件推导产品信息。
- 扫码校验不再要求产品名称、产品规格、BOM 编号、BOM 版本、物料名称、物料规格、数量或分类。
- 不再根据 BOM 物料清单校验导入的物料编码，只要求物料编码存在且不为空。
- 从产品配置和作业记录页面移除 BOM 字段。
- 从面向操作员的界面移除 BOM 发布、启用、作废、归档、历史版本比较、源文件和 SHA-256 控件。
- 新建产品配置时不再关联 BOM。
- 为兼容已有数据，保留数据库中现有 BOM 表、已导入的 BOM 行以及历史产品配置引用。
  本次修改不物理删除历史 BOM 数据。
- 正常操作流程不再读取或显示历史 BOM 数据。只有在备份、恢复和历史数据兼容性分别验证通过后，
  后续数据库清理才可以删除闲置的 BOM 代码。

### 优先级 2：简化产品配置导入

- 接受只包含“站位编码”和“物料编码”的两列站位表。
- 在校验和保存过程中，根据站位主数据自动识别“设备编码”。
- 为兼容旧文件，继续接受原来的三列表格。
- 旧表格如果提供了“设备编码”，但与该站位实际所属设备不一致，则拒绝导入该行。
- 更新生成的 Excel 模板和帮助说明，解释两种支持的格式。
- 导入页面要求填写非空的产品编码和配置版本。
- 产品编码与配置版本重复时，显示明确的中文提示。
- 至少需要一条有效的站位与物料对应关系。
- 通过一次操作完成产品配置的导入和启用。

### 优先级 3：简化产品配置编辑

- 用户只编辑站位编码和物料编码。
- 系统自动识别并保存所属设备。
- 需要时，可以将识别出的设备作为只读信息显示，但不要求用户编辑。
- 只根据站位编码判断是否重复。
- 从产品配置编辑和配置列表中移除 BOM 选择器及 BOM 信息。

### 优先级 4：简化扫码页面

- 最近扫码记录默认隐藏设备列，为笔记本屏幕上的站位和物料列留出更多宽度。
- 在作业详情、审计数据、扫码记录和 CSV 导出中继续保留设备信息。
- 扫描站位后显示确认信息，例如“当前站位 F-01 · 所属设备 SMT-01”。
- 等待扫描物料时，提供明确的“重新扫描站位”操作，用于纠正已选择的站位。
  不能把任意物料码自动当作站位切换，因为两类编码可能发生重叠。

### 优先级 5：增加面向操作员的短作业号

- 在表示一次生产扫码作业的界面中，将“运行号”改为“作业号”。
- 在扫码页面、作业列表和作业摘要中显示类似 `0720-003` 的每日短序号。
- 保留类似 `RUN-20260720-101856-XXXXXXXX` 的原始编号，作为不可变的内部 `run_id`；
  不修改历史扫码记录、运行状态、审计行或外键关系。
- 明确保存短作业号，不能只在界面上截断完整运行编号。
- 在 SQLite 中以原子方式分配每日序号，确保程序重启后不会重复使用作业号。
- 在作业详情、提示信息、审计数据和 CSV 导出中保留完整内部运行编号。
- 导出时同时提供“作业号”和“内部运行编号”，保证现场使用的短编号能够追溯到准确的运行记录。
- 在不修改历史 `run_id` 的前提下，为历史作业提供稳定的短显示编号。

### 优先级 6：内部代码清理

- 确认没有外部调用后，删除已经不再使用的“先选设备再扫码”辅助代码。
- 集中管理全局站位识别逻辑，使导入、产品配置编辑和扫码使用一致的校验行为及错误提示。
- 只有在产品配置直接导入流程已有完整测试覆盖后，才删除依赖 BOM 的应用服务和界面连接代码。
- 在单独批准数据库迁移删除 BOM 前，继续保证旧 BOM 数据可以读取。
- 集中管理短作业号的分配和格式化，不能由各个界面组件自行生成显示字符串。

## 笔记本界面的进一步简化建议

以下建议是在笔记本尺寸窗口和最大化布局下检查所有现有页面后记录的。它们是在已经确认的
“站位优先、操作流程无 BOM、短作业号”方向上继续优化。除非明确实施，否则这些条目只是后续
建议，不会自行改变业务数据或程序行为。

当前主要问题不是全局字体过大，而是界面同时展示了过多概念、低频操作和固定高度区域；
窄窗口又继续把左右两栏压缩在一起。应保留清晰可读的中文字体，通过明确的信息层级、响应式布局、
按状态显示操作以及删除重复内容来解决密度问题。

### 界面优先级 0：修复容易误解或不安全的界面状态

- 扫码数量为 `0 / 0` 时显示空的灰色进度条。在作业未开始前，不得显示循环运动的绿色进度动画。
- 移除扫码中央提示区域和最近扫码记录区域之间互相冲突的固定最小高度及最大高度。
  调整窗口或展开最近记录后，中央提示不得被纵向裁切。
- 切换设备时，先清空站位编辑框，再刷新站位列表。只有选择当前设备下的站位后，才填充站位字段，
  防止误编辑或保存上一个设备遗留的站位编码。
- 保存站位前，再次校验该站位是否属于当前选择的设备。
- 将所有显示给用户的审计操作和系统操作主体转换为中文，包括 `DELETE`、`ARCHIVE` 和
  `SYSTEM_MIGRATION`。
- 将审计修改内容显示为易读的中文字段和值差异。原始 JSON 只能通过折叠的“查看原始数据”操作查看。

### 界面优先级 1：精简主导航和常驻界面元素

- 日常导航只保留“扫码作业”“作业记录”“产品配置”和“设备与站位”。
- 将“生产运行”改名为“作业记录”，与面向操作员的“作业号”用词保持一致。
- 将产品配置导入作为“产品配置”页面中的主要“导入新配置”操作，不再单独占用一个导航入口。
- 将“审计日志”移动到低频使用的“设置”或“更多”中。
- 将操作员控制保留在左下角侧边栏，把当前操作员和“切换操作员”合并为一个紧凑区域。
- 从普通页面移除常驻的整行诊断信息。数据库、语音或扫码枪正常时不显示诊断；异常时再提示，
  正常诊断详情放入设置页面。
- 将“恢复默认布局”移入设置，并改名为更明确的“重置表格列宽和分栏”。
- 将永久显示的“就绪”和成功提示条改为短时间显示的通知。校验错误应显示在引发错误的字段或操作附近。

### 界面优先级 2：让扫码页面专注于当前任务

- 作业开始前，只显示一行紧凑设置内容：产品配置选择框和一个主要的“开始作业”按钮。
  不要在多个标题和空状态卡片中重复显示已选产品和“未开始”状态。
- 作业开始后，锁定已选择的产品配置，用短作业号、进度和一个醒目的动态下一步提示替换设置说明。
- 使用“请扫描站位码”等任务提示；识别站位后显示“当前站位 F-01 · 所属设备 SMT-01”，
  再提示“请扫描物料码”。
- 笔记本上的中央提示区域保持紧凑，高度约为 160 至 200 个逻辑像素，其余纵向空间分配给
  最近扫码记录表格。
- 空间允许时，默认显示最近 5 至 8 条扫码记录，不要求用户先展开。
- 最近扫码记录使用“时间”“站位”“要求物料”“扫码物料”和“结果”列。默认隐藏设备列；
  只有实际发生重复时才显示重复信息。
- 当天的扫码记录只显示 `HH:mm:ss`，不在每一行重复完整日期。
- 正常扫码枪操作中隐藏“手动提交”。只有进入明确的管理员或“手动输入模式”后才显示，
  供键盘调试使用。
- 扫码结果同时使用文字和颜色。NG 提示必须明确写出要求物料和实际扫码物料，且不能长时间阻塞
  下一次扫码。

### 界面优先级 3：管理页面应重新排版，不能只压缩列宽

- 宽屏可以保留列表与详情左右分栏。笔记本内容宽度较小时，应让列表占满宽度，并通过详情抽屉
  或独立详情页打开所选记录，不能继续把左右两栏挤在一起。
- “作业记录”和“产品配置”都应用上述响应式布局；不能只依赖缩小列宽、横向滚动条或已保存的
  分栏比例。
- 作业列表默认显示“作业号”“产品/配置版本”“状态”“进度”和“开始时间”。
  操作员、完整内部运行编号、结束时间、中断说明和追溯信息放入所选作业详情。
- 默认作业筛选只显示一个关键词框、状态和类似“近 7 天”的日期快捷选项。
  只有用户选择自定义日期范围后，才展开具体开始和结束时间。
- 从作业筛选区移除“转到扫码开始”，因为主导航已经可以进入扫码页面。
- 将“恢复作业”“导出 CSV”和“中断作业”放在所选作业详情中，只显示当前状态实际可用的操作。
- 扫码记录只保留一个入口；不能同时显示扫码记录页签和重复的“查看扫码记录”按钮。

### 界面优先级 4：简化产品配置和基础数据维护

- 移除 BOM 后，产品配置列表使用“产品编码”“配置版本”“状态”“站位数”和“更新时间”列。
- 产品配置详情以“站位编码”和“物料编码”为主要可编辑值。需要时将所属设备显示为只读信息。
- 每种产品配置状态只显示一个主要操作。草稿状态允许编辑站位行和“保存并校验”；
  已启用或历史版本只读，主要提供“复制为新版本”。
- 保存或导入时自动进行产品配置校验，正常操作不再需要单独点击生命周期校验按钮。
- 在一个页面完成新配置导入：产品编码、配置版本、一个 Excel 文件、校验摘要和一个
  “导入并使用”操作。保留模板下载和详细校验错误，但不恢复多步骤向导。
- 设备和站位维护应有明确的查看、新建和编辑模式。不能同时把新建、保存、启用、停用作为同等
  醒目的操作展示。
- 已启用对象只显示可用的“停用”；已停用对象只显示“启用”。
- 批量创建站位作为低频操作，默认折叠或放入“更多操作”。
- 默认隐藏低频使用的“已引用”站位列。如果生产流程不依赖站位名称，则将其作为可选的次要信息。
- 全局唯一站位编码发生冲突时，错误信息必须指出其所属设备，例如
  “站位 F-01 已属于设备 SMT-01”。

### 界面优先级 5：建立克制且统一的视觉与交互规范

- 不通过全局缩小字体来塞入更多内容。正文、表格和按钮文字保持约 14 至 16 个逻辑像素；
  通过减少字段和响应式布局解决空间问题。
- 每个任务区域只使用一个蓝色主要按钮。次要操作使用中性描边样式；破坏性样式只用于当前所选对象。
- 减少多层浅色卡片和边框。优先使用一个主要白色内容区域、标题和细分隔线；
  只有适合分组的摘要或状态才使用卡片。
- 永久不可用的操作应隐藏。操作只是暂时不可用时，应在控件附近说明原因。
- 表格行高保持约 36 至 40 个逻辑像素。应优先安排重要列，不能把所有列都压缩到最小宽度。
- 状态同时使用文字和颜色，保证色觉较弱的用户也能理解。
- 保留扫码枪键盘行为和清晰的输入焦点。可安全使用时增加 `Ctrl+F` 聚焦当前页面筛选框、
  `Esc` 取消或清空等普通键盘操作。
- 中断作业、停用设备或停用配置等状态操作必须显示简短且明确到具体对象的确认提示。
  普通搜索和保存操作不要增加确认对话框。
- 刷新后，只要对象仍然存在，就保留当前选择行。
- 空状态应提供明确行动，例如“还没有可用的产品配置”并附带一个“导入产品配置”操作，
  不能只显示大块空白区域。

### 笔记本界面验收矩阵

- 在 Windows 125% 显示缩放下，检查 `1366x768` 最大化和普通窗口布局。
- 在 Windows 125% 和 150% 显示缩放下，检查 `1440x850` 最大化和普通窗口布局。
- 每个页面都要分别使用空数据、真实数据、较长的产品/物料/站位值以及至少 100 行表格数据检查。
- 检查扫码状态：“未开始”“等待站位”“等待物料”“正确”“NG”“已完成”和“已中断”。
- 确认从大显示器调整到笔记本窗口时，不会遗留被裁切的提示、错误分栏比例、无法使用的已保存列宽
  或被隐藏的主要操作。
- 笔记本高度允许时，最近扫码记录至少显示 5 行清晰可读的数据。
- 日常业务表格的默认列不得依赖横向滚动。审计或以导出为目的的高级数据允许横向滚动。
- 所有被截断的值必须能在所选记录详情或提示信息中查看，主要操作流程不能依赖鼠标悬停。
- 检查扫码枪按 `Enter` 提交、页面切换后扫码输入自动获得焦点、明确的站位重扫操作以及无需鼠标
  从 NG 状态恢复。
- 破坏性状态操作在确认前必须明确写出受影响的对象和结果。

## 兼容性要求和不实施内容

- 不删除设备，也不删除设备与站位之间的所属关系。
- 不从产品配置关系、作业快照、扫码记录、审计日志或导出中删除 `device_code`。
- 不仅为了简化界面显示就替换现有复合键；这种数据库迁移风险高，对操作员没有明显收益。
- 发现站位唯一性冲突时，不得自动合并、改名或删除旧站位。
- 在操作流程简化期间，不删除 BOM 数据表或已导入的 BOM 数据。
- 不删除或缩短现有内部 `run_id`；它仍然是长期稳定的技术主键。
- 短作业号不能成为审计和导出中唯一可用的编号。

## 验收标准

- 当所有站位都存在、处于启用状态，并且分别只属于一个已启用设备时，新的两列 Excel 可以成功导入。
- 导入页面只要求“产品编码”“配置版本”和一个包含站位与物料关系的两列 Excel，不要求 BOM 文件。
- 正常导入页面不要求操作员填写 Excel 工作表名称。Excel 只有一个页签时自动导入；存在
  `Worksheet` 时优先读取；只有多个页签确实无法判断时才要求用户明确选择。
- 现有三列 Excel 无需修改即可继续导入。
- 旧 Excel 中设备与站位不匹配时，应拒绝导入并显示源文件行号。
- 物料编码为空、站位编码重复、站位未知或停用、配置为空、产品编码与配置版本重复时，
  均应拒绝并显示明确的中文提示。
- 即使物料编码从未出现在 BOM 中，也能正常导入并用于扫码校验。
- 正常导航和导入流程中不再出现“BOM 管理”和 BOM 生命周期操作。
- 界面不再显示 BOM 数据后，引用历史 BOM 的现有产品配置和生产历史仍然可以正常读取。
- 从可编辑输入中删除设备列后，产品配置编辑仍应保存相同的内部
  `(device_code, station_code) -> material_code` 对应关系。
- 扫码页面只把站位和物料作为操作员需要扫描的两个步骤。
- 校验物料前，界面显示当前站位和系统自动识别的所属设备。
- 扫描站位后，中央区域完整显示“请扫描物料码”和当前站位、所属设备；在所有支持的窗口尺寸、
  显示缩放比例和扫码状态下，文字不得被其他控件覆盖或裁切。
- 扫码页面的步骤、站位、设备及物料信息不得重复占用多行空间；优化后的中央提示区域不能减少
  最近扫码记录在笔记本窗口中的可见行数。
- 新作业获得短小、唯一、每日顺序递增的“作业号”，同时保留完整内部 `run_id`。
- 程序重启后新建作业，不得重复使用已有短作业号。
- 作业详情、审计数据和 CSV 导出都能够将每个短作业号准确对应到唯一的完整内部运行编号。
- 历史作业显示稳定的短作业号，且不修改其已保存的完整运行编号。
- 最近扫码记录在 `1366x768` 或 `1440x850` 笔记本屏幕上能够显示更多有效内容，
  同时在作业详情和 CSV 导出中保留完整追溯信息。
- 作业详情默认不显示与左侧列表、顶部摘要卡、标题或底部操作区重复的信息块；删除该显示区域后，
  操作员、结束时间、中断原因和内部运行编号仍可通过审计记录、CSV 导出或折叠详情完整追溯。
- 删除重复作业信息块后，释放出的纵向空间应分配给站位进度和扫码记录表格。
- 单元测试、内存 SQLite 集成测试、数据库迁移、导入兼容、备份恢复兼容、CSV 导出以及
  Qt 离屏界面测试应覆盖新流程和旧数据兼容路径。
